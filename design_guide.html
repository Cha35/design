02 Badge
사용자에게 정보를 강조하거나 특정 요소에 주의를 끌기 위해 아이콘 또는 텍스트에 추가하는 요소
타입
타입설명용도
Badge-basic숫자 표시 (1자리 / 10자리)장바구니 개수, 알림 개수
Badge-dot점(dot)으로 알림 표시새 정보 포함 알림
Customized badge커스텀 텍스트특수 케이스
Badge-basic 규칙
  • 색상: CTA / Primary 컬러 사용
  • 1자리 / 10자리(99+) 구성
  • 0개는 미노출 처리
Badge-dot 규칙
  • 해당 영역에 새 정보가 포함될 때 사용
03 Bread Crumbs
현재 위치의 맥락을 보여주는 내비게이션 요소
규칙
  • 홈부터 상위 카테고리(2depth) → 하위 카테고리(3depth) → 현 위치 순서
  • 클릭 시 해당 페이지로 이동 가능 (서비스 내 이탈률 감소 목적)
  • 현 위치는 볼드 처리
  • 최대 4depth까지 표현 | 구분자: > (꺽쇠)
04 Chip
정보, 선택, 콘텐츠 필터링, 특정 액션 유도 컴포넌트. 리스트 필터링 기능에 한정하며, 기능에 따라 사용처 한정
타입
타입설명
Chips-basic최소 2가지 옵션, 기본 필터링 목적
Chips-option다수 선택지, 선택 시 나열 표시
상태
  • Default (미선택): 아웃라인 스타일
  • Select (선택됨): 검정 배경 + 흰 텍스트
Scroll 처리
  • 칩이 디바이스 너비 초과 시 가로 스와이프 적용
  • 좌측 아이콘 Fixed, 나머지 스크롤
  • Fixed 아이콘 뒤 배경: linear-gradient(90deg, #FFF 80.15%, rgba(255,255,255,0.00) 100%)
05 Date Picker
규칙
  • 시작 시점 + 종료 시점 선택 가능 (월 기준 / 일 기준 모두 가능)
  • 종료 시점은 현재 기준으로 적용, 사용자 직접 수정 가능
  • 종료 시점은 현재보다 미래 설정 불가
  • 전체 시점 미선택 시 버튼 비활성화 → 모두 선택 후 버튼 활성화
컴포넌트
  • 드럼롤(스크롤) 방식: 일 선택용
  • 드럼롤 방식: 월 선택용 (검정 배경 선택 상태)
  • 달력 방식: 날짜 전체 보기 (선택일 파란 원 표시, 일요일 빨강, 토요일 파랑)
06 Divide-Line (구분선)
디자인 시 구분선 사용 지양 → 간격 조절(padding)로 영역 구분 우선. 영역 구분이 애매할 때만 사용
굵기 타입
  • 얇은 선: 같은 레벨 콘텐츠 구분
  • 굵은 선: 다른 섹션/영역 구분
07 Dropdown
타입
타입설명
Dropdown필터링하는 드롭다운
Dropdown-Layer동작으로 트리거되는 옵션 목록 표시
Dropdown 상태
  • 기본 (닫힘): 텍스트 + chevron-down
  • 열림: 텍스트 + chevron-up + 옵션 리스트
  • 선택됨: 선택된 옵션 텍스트 표시
  • 오류: 빨간 테두리
  • 비활성화: 회색 처리
공통 규칙
  • 너무 긴 드롭다운 메뉴 지양
  • Dropdown-Layer: 페이지 선택 옵션 Default MAX 168 고정, 영역 내 스크롤 진행
08 Empty_case (빈 상태)
구성 요소
  • 아이콘 (느낌표 또는 원형 아이콘)
  • 강조 타이틀 / 서브 타이틀 / CTA 버튼 (선택)
스타일 타입
타입특징
타입 A느낌표(!) 아이콘 + 타이틀 + 서브 + 버튼
타입 B원형 아이콘 + 타이틀 + 서브 + 버튼
CTA 버튼
  • 색상: Primary (파란색)
  • 라벨: "확인" 또는 상황에 맞는 액션
09 Error_case (에러 상태)
에러 타입
타입문구CTA
통신 오류 (GNB 없음)"화면을 불러오지 못했어요 / 네트워크 연결 상태를 확인한 후 다시 시도해 주세요."없음
불러오기 실패 (단순)"일시적으로 화면을 불러오는데 어려움이 있어요 / 잠시 후 다시 시도해 주세요."없음
불러오기 실패 + 재시도동일다시 시도
불러오기 실패 + 복합 CTA동일메인으로 이동 + 다시 시도
GNB 표시 규칙
  • GNB 경우 화면에 따라 노출 유무 결정
  • 빈 화면의 앱 바는 에러에서도 노출
  • 스토어 예외: 전체 템플릿하기 때문에 GNB 없이 노출
10 Title_img_sub_case
규칙
  • Navigation과 타이틀 사이 간격: 80px 고정
  • 타이틀이 길 경우 2줄로 진행
  • 이미지 비율: 7:3 비율 내에서 활용
  • 광고는 on/off 가능
레이아웃 구성 순서
  1. Navigation (헤더)
  2. 타이틀 (최대 2줄)
  3. 이미지 영역 (7:3 비율)
  4. 서브 텍스트 (선택)
  5. 광고 배너 (선택, on/off)
  6. CTA 버튼 ("확인")

  • 첫 화면 로드나 새로 고침이 아닌 한 번에 두 이상의 항목 로딩 지양

브랜드 로딩폴리볼 로고 (어두운 배경)
광고 로딩"광고가 끝나면 보상 혜택이 적용돼요 / 광고를 불러오고 있어요." + dot 인디케이터
12 Navigation
사용자가 필요한 것을 찾도록 도와주며 현재 위치를 알려주는 컴포넌트. 2가지 타입: Navigation-bottom / GNB 헤더
GNB-header
타입구성
메인 헤더 (라이트)PolyBall 로고 + 알림 배지 + 알림 아이콘
메인 헤더 (다크)PolyBall 로고 + 알림 아이콘 + 설정 아이콘
서브 헤더 (기본)← 뒤로가기 + 페이지 타이틀
서브 헤더 (우측 액션)← 뒤로가기 + 페이지 타이틀 + 우측 텍스트 링크
Navigation-bottom
  • 최대 5개 아이콘으로 구성, 화면 최하단 고정
  • 폴리볼 현재 탭 구성: 승부예측 / 응모 / 마이 (3탭)
  • 선택된 탭: 아이콘 채움(fill) + 텍스트 볼드 + 하단 인디케이터 바
  • 미선택 탭: 아이콘 아웃라인 + 텍스트 일반
  • 674 해상도 기준 탭바 높이 별도 적용

14 Selection Controls
Radio Button
  • 상호 배타적인 두 개 이상의 옵션, 그 중 하나만 선택
  • Default로 하나는 무조건 선택되어 있어야 함
  • 상태: unselected / selected
Checkbox
  • 한 개 또는 다수의 독립적인 옵션, 선택하지 않을 수도 있음
  • 하나 또는 여러 개 동시 선택 가능
  • 한 개인 경우 기능을 on/off 하는 용도
  • 상태: unchecked / checked / disabled
Toggle-Switch
  • 상호 배타적인 옵션이 있어 각 선택에 따른 효과 전환이 필요한 경우
  • 상태: off / on (4단계: 기본off → 중간off → 중간on → 완전on)

17 Stepper
아이콘 선택으로 입력값을 조절하는 요소. 타입: plus-minus → − [숫자] + 형태
규칙
  • 최솟값에서 − 버튼 비활성화 (예: 1이 최솟값이면 − 비활성)
  • 최댓값에서 + 버튼 비활성화 (예: 9999가 최댓값이면 + 비활성)
  • 입력 필드 클릭하여 값을 직접 입력 및 변경 가능

  • 전체 배경: Polyball BLUE (#1F30E9) 풀스크린

  • 2열 그리드 팀 카드: 팀명(상단) + 마스코트(하단) + 팀컬러 배경

  • 선택됨: 파란 계열 배경 + 파란 테두리
  • 하단 CTA: "응모하기" (Primary 버튼, 풀 width)
